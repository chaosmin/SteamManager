# SteamManager UI/UX Design Plan — v0.4.0

2026-07-15

## Scope

UI/UX changes for the v0.4.0 architecture: game queue panel, dual concurrent game cards, reference player configuration, updated achievement table, new `Scheduled` game state. Builds on the Steam-dark design system from `20260707.md`.

---

## Design System Additions

### New Status Chip Colors

| Status | Color | Usage |
|--------|-------|-------|
| `Idle` | Grey (default) | Game in queue, not yet started |
| `Playing` | `#66C0F4` Steam Blue | Game actively idling |
| `Scheduled` | `#3B82F6` Info Blue | MCT reached, achievements still pending |
| `Completed` | `#22C55E` Green | MCT reached + all achievements unlocked |

### New Achievement Status Chips

| Status | Color | Condition |
|--------|-------|-----------|
| `Unlocked` | `#22C55E` | `is_unlocked = 1` |
| `Scheduled` | `#3B82F6` | `scheduled_unlock_at IS NOT NULL AND is_unlocked = 0` |
| `Locked` | Grey | `scheduled_unlock_at IS NULL AND is_unlocked = 0` |

## P0 — Dashboard

### Queue Control Panel

Inserted between the global stat cards and the Currently Playing section.

```
┌─────────────────────────────────────────────────────────┐
│  Queue                               [▶ Start Queue]    │
│  ─────────────────────────────────────────────────────  │
│  ⠿  🖼  Factorio          [Playing]   452h / 600h  [×]  │
│  ⠿  🖼  Portal 2          [Playing]    12h /  40h  [×]  │
│  ⠿  🖼  Hades             [Idle]        0h /  80h  [×]  │
│  ⠿  🖼  Celeste           [Idle]        0h /  20h  [×]  │
│                                                         │
│  Add games via Refresh on the game detail page.         │
└─────────────────────────────────────────────────────────┘
```

**Drag-and-drop rules:**
- `Playing` items: drag handle disabled (`pointer-events: none`), subtle left border `#66C0F4`
- `Scheduled` items: drag handle disabled, left border `#3B82F6`
- `Idle` items: fully draggable
- On drop → `GameQueueService.ReorderAsync` (optimistic local update)

**Start Queue button:**
- Disabled when no `Idle` games in queue
- On click → fills up to 2 Playing slots from top of queue

**Implementation (MudBlazor):**
```razor
<MudDropContainer T="GameQueueEntry" Items="_queue"
                  ItemDropped="OnItemDropped">
    <ChildContent>
        <MudDropZone T="GameQueueEntry" Identifier="queue">
            @foreach (var item in _queue)
            {
                <MudPaper Class="queue-row d-flex align-center gap-3 pa-2 mb-1"
                          Style="@GetQueueRowStyle(item)" Elevation="0">
                    <MudIcon Icon="@Icons.Material.Filled.DragIndicator"
                             Class="@(item.IsLocked ? "drag-disabled" : "drag-handle")" />
                    <MudImage Src="@item.HeaderUrl" Width="48" Height="22"
                              ObjectFit="ObjectFit.Cover" Class="rounded" />
                    <MudText Typo="Typo.body2" Class="flex-grow-1">@item.Name</MudText>
                    <MudChip T="string" Size="Size.Small"
                             Color="@GetStatusColor(item.Status)">@item.Status</MudChip>
                    <MudText Typo="Typo.caption"
                             Style="color:rgba(255,255,255,0.45); min-width:80px; text-align:right">
                        @FormatQueuePlaytime(item)
                    </MudText>
                    <MudIconButton Icon="@Icons.Material.Filled.Close"
                                   Size="Size.Small" Disabled="@item.IsLocked"
                                   OnClick="@(() => RemoveFromQueue(item.GameId))" />
                </MudPaper>
            }
        </MudDropZone>
    </ChildContent>
</MudDropContainer>

<MudButton Variant="Variant.Filled" Color="Color.Primary"
           StartIcon="@Icons.Material.Filled.PlayArrow"
           Disabled="@(!_queue.Any(q => q.Status == "Idle"))"
           OnClick="StartQueue" Class="mt-2 w-100">
    Start Queue
</MudButton>
```

```css
.queue-row         { border-radius: 6px; background: rgba(255,255,255,0.04); }
.queue-row:hover   { background: rgba(255,255,255,0.07); }
.drag-handle       { cursor: grab; color: rgba(255,255,255,0.3); }
.drag-disabled     { cursor: default; color: rgba(255,255,255,0.1); }
```

## P1 — GameDetail: Reference Player Config

### Placement

Collapsible panel between the game header row and the achievement table.

```
┌──────────────────────────────────────────────────────────────┐
│ Reference Player  [●  steamhunters.com/profiles/...]  [▼]   │
│ ──────────────────────────────────────────────────────────  │
│ SteamHunters Player URL                                      │
│ [ https://steamhunters.com/profiles/76561198012345678  ]     │
│                                                              │
│ □  Override burst detection                                  │
│    Allow Refresh even if player unlocked ≥ 5 achievements    │
│    at the same timestamp.                                    │
│                                                              │
│                          [Save]    [Remove Player]           │
└──────────────────────────────────────────────────────────────┘
```

**Panel states:**

| State | Visual |
|-------|--------|
| No player | `MudAlert` Info — "No reference player configured. Scheduled unlocking will use estimated intervals." |
| Player set, healthy | Green dot + truncated URL in panel header |
| Burst detected (last Refresh blocked) | `MudAlert` Warning — "Player data has N achievements at the same timestamp. Enable override to proceed." |
| SteamHunters unavailable | `MudAlert` Error — "Could not fetch player data. Check URL or try again." |

```razor
<MudExpansionPanel Dense="true">
    <TitleContent>
        <div class="d-flex align-center gap-2">
            <MudIcon Icon="@Icons.Material.Filled.Person" Size="Size.Small" />
            <MudText Typo="Typo.body2">Reference Player</MudText>
            @if (_hasPlayer)
            {
                <MudIcon Icon="@Icons.Material.Filled.Circle" Size="Size.Small"
                         Style="color:#22C55E; font-size:8px" />
                <MudText Typo="Typo.caption"
                         Style="color:rgba(255,255,255,0.5)">@_playerUrlShort</MudText>
            }
        </div>
    </TitleContent>
    <ChildContent>
        @if (_burstDetected)
        {
            <MudAlert Severity="Severity.Warning" Dense="true" Class="mb-3">
                Player data has @_burstCount achievements at the same timestamp.
                Enable override to allow Refresh.
            </MudAlert>
        }
        <MudTextField @bind-Value="_playerUrl" Label="SteamHunters Player URL"
                      Variant="Variant.Outlined" FullWidth="true" Class="mb-3"
                      Adornment="Adornment.Start"
                      AdornmentIcon="@Icons.Material.Filled.Link" />
        <MudCheckBox @bind-Value="_overrideBurst" Dense="true" Class="mb-3">
            <span>Override burst detection
                <span style="color:rgba(255,255,255,0.45); font-size:0.75rem; display:block">
                    Allow Refresh when ≥ 5 achievements share the same unlock timestamp
                </span>
            </span>
        </MudCheckBox>
        <div class="d-flex gap-2">
            <MudButton Variant="Variant.Filled" Color="Color.Primary"
                       OnClick="SavePlayer">Save</MudButton>
            @if (_hasPlayer)
            {
                <MudButton Variant="Variant.Outlined" Color="Color.Error"
                           OnClick="RemovePlayer">Remove</MudButton>
            }
        </div>
    </ChildContent>
</MudExpansionPanel>
```

## P1 — GameDetail: Achievement Table

### Column Layout

| Column | Width | Notes |
|--------|-------|-------|
| Icon | 48 px | 40×40, unlocked color / locked grey |
| Name + Description | flex | Primary + caption secondary |
| Scheduled At | 160 px | Local timezone; `—` if null |
| Global % | 100 px | Mini `MudProgressLinear` + caption |
| Status | 110 px | Chip: Unlocked / Scheduled / Locked |
| Time | 120 px | See states below |

**Time column states:**

| Condition | Display | Style |
|-----------|---------|-------|
| `is_unlocked = 1` | `UnlockedAt` local time | Green `#22C55E` |
| Scheduled, future | `in Xh Ym` | Blue `#3B82F6` |
| Scheduled, past, pending | `Pending ●` (pulsing dot) | Orange `#F59E0B` |
| Not scheduled | `—` | Muted |

**Row visual treatment:**
```css
.row-unlocked  { opacity: 0.55; }
.row-next      { border-left: 3px solid #3B82F6;
                 background: rgba(59, 130, 246, 0.05); }
.row-pending   { border-left: 3px solid #F59E0B;
                 background: rgba(245, 158, 11, 0.04); }

@keyframes pulse-dot {
    0%, 100% { opacity: 1; }
    50%       { opacity: 0.3; }
}
.pending-dot { animation: pulse-dot 1.5s ease-in-out infinite; }
```

**Sort default:** `scheduled_unlock_at` ASC, nulls last. Toggle-sort by Global % available.

## P2 — Games Page

### Refresh Button State Machine

| Game state | Button shown |
|------------|-------------|
| No reference player | `[Refresh]` disabled; tooltip "Configure a reference player in game detail first" |
| Has player, never refreshed | `[Refresh]` enabled (outlined primary) |
| In queue (`Idle`) | `[Re-Refresh]` + queue position badge on card |
| `Playing` | `[Stop]` only |
| `Scheduled` | No action button; status chip only |
| `Completed` | `[Force Refresh]` (outlined warning) |

### Queue Position Badge

When a game is queued and `Idle`, show its queue position on the card thumbnail:

```razor
<MudBadge Content="@_queuePos.ToString()" Color="Color.Primary"
          Overlap="true" Bordered="true">
    <MudImage Src="@HeaderUrl" ... />
</MudBadge>
```

Badge position: top-left corner. Styling: `#66C0F4` background, white text, `font-size: 0.7rem`.

## Revisions — 2026-07-15 (Session 2)

### Games Page: Game Card

**Queued chip:** `#eab308` amber yellow background, `#000` text — replaces the grey "Idle" chip when game is in queue.

**Thumbnail fallback:** When Steam header image 404s, show a `position:relative` container with:
- Background: `linear-gradient(135deg,#1a2332,#0d1b2a)`
- Centered placeholder: `VideogameAsset` icon (32 px, `rgba(255,255,255,0.15)`) + game name (`rgba(255,255,255,0.3)`, 0.7rem)
- Actual `<img>` absolutely positioned on top; hidden via `onerror="this.style.display='none'"` when it fails

**Button state machine (final):**

| State | Button | Icon | Color |
|-------|--------|------|-------|
| No ref player | hidden | — | — |
| Has player, not queued | Add to Queue | `QueuePlayNext` | Success green |
| Has player, queued (Idle) | Remove from Queue | `RemoveFromQueue` | Warning orange |
| Playing | Stop | `Stop` | Error red |
| Scheduled | chip only | — | — |
| Completed | no action | — | — |

### GameDetail: Achievement Table

**Sort order (final):**
- Groups (OrderBy): overdue scheduled (0) → future scheduled (1) → unscheduled locked (2) → unlocked (3)
- Within groups: pending → `.ThenBy(Ticks ASC)` (earliest first); unlocked → `.ThenBy(MaxValue − UnlockedAt.Ticks)` (most recent first via inverted ascending trick)

**Time column:** `yyyy-MM-dd HH:mm` format (local time). Column min-width 165 px.

**Loading overlay:** `MudOverlay` with `DarkBackground` + `MudProgressCircular` covers full page during Save + scheduling.

### Dashboard: Refresh Behavior

All mutating actions (Start Queue, Stop Queue, Remove from Queue) now call `ReloadAsync()` immediately after:
```csharp
private async Task ReloadAsync()
{
    _games = await Db.Games.Include(g => g.Achievements).AsNoTracking().ToListAsync();
    ComputeStats();
    _queue = await QueueService.GetQueueAsync();
}
```
1-minute background timer also refreshes `_maxSlots` from DB on each tick.

### Settings: Queue Settings Card

New card in right column below Background Sync:
- `MudNumericField` bound to `MaxConcurrentGames` (min 1, max 32)
- Saves to `steam_config.max_concurrent_games`
- Loaded on page init from DB

### Completed Games

- `GameStatus.Completed` games: no play/requeue buttons anywhere
- Filtered from `GetQueueAsync()` (hidden from Dashboard queue list)
- Status chip top-right only, no label below card in Games page
- `UnlockSchedulerService` calls `RemoveFromQueueAsync` when marking game Completed
