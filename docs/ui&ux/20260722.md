# SteamManager UI/UX Design Plan — v0.4.1

2026-07-22

## Scope

UI/UX changes for v0.4.1: Games page filter system redesign, GameDetail page full layout overhaul. Builds on the Steam-dark design system from `20260707.md` and `20260715.md`.

---

## 1. Games Page — Filter Bar Redesign

### Problem

Previous design used `MudSwitch` toggles for property filters and had no status-based filtering. Switches lacked visual hierarchy, couldn't represent "no filter" vs "filter active" clearly, and required extra vertical space.

### Solution

Two-row chip layout:

```
Row 1: [Search input ──────────────] [Sort dropdown]
Row 2: [All] [Playing] [Queued] [Scheduled] [Completed]   [Has Achievements] [100% Done] [Card Drops]   [Clear All ×]
```

**Principles applied:**
- Single-select status chips (mutually exclusive) — only one state context visible at a time
- Multi-select property chips — composable with status filter
- `Clear All` only appears when a non-default filter is active (reduces visual noise at rest)
- Result count above grid gives immediate feedback on filter effectiveness
- Chip active state uses `Color="Color.Primary"` for clear affordance

### Queued state definition

"Queued" chip matches games where `Status == Idle` AND the game's ID is present in the queue list. This is a derived state — no `GameStatus.Queued` enum value exists. Filter fetches queue IDs at load time and includes them in the filter predicate.

### Sync for Completed games

Previously hidden. Now shown — users may want to pull latest Steam playtime after external edits or to confirm completion status.

## 2. GameDetail Page — Full Layout Redesign

### Problem

Previous layout: flat card stack with no visual hierarchy. Hero area lacked identity. Stats were buried in text fields. Store description/tags missing.

### Design direction: Steam-dark editorial

Matches the aesthetic of the Steam store page for the game itself. Dense information, high visual contrast, strong use of the header art.

### Hero Banner

```
┌────────────────────────────────────────────────────────────┐
│  [Steam header image 460:215, object-fit: cover]           │
│  ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  │
│  gradient overlay (transparent → rgba(0,0,0,0.85) bottom)  │
│                                                            │
│  GAME TITLE                    [status chip]               │
│  [Steam Store ↗]  [SteamHunters ↗ — if total > 0]         │
└────────────────────────────────────────────────────────────┘
```

- Aspect ratio locked at 460/215 (Steam header standard)
- Gradient: `linear-gradient(to bottom, transparent 40%, rgba(0,0,0,0.85))`
- Title: `MudText Typo.H5`, white, absolute positioned bottom-left
- Buttons: `Variant.Outlined Size.Small`, grouped beside title
- SteamHunters button conditionally rendered: `@if (_total > 0)`

### Stats Strip

Horizontal `MudPaper` row below hero, 5 stat cards:

| Card | Value source |
|------|-------------|
| Playtime | `game.SteamPlaytimeAtRefresh` → `FormatMinutes()` |
| Target | `game.TargetMinutes` → `FormatMinutes()` |
| Achievements | `_unlocked / _total` |
| App ID | `game.AppId` |
| Last Synced | `game.AchievementsCachedAt` → `FormatTime()` |

Each card: small muted label on top, larger white value below. Responsive: wraps on narrow viewports via `MudGrid`.

### Description + Tags Section

Loaded asynchronously after initial render via `InvokeAsync(StateHasChanged)`. Does not block page display.

```
[Short description text — muted color, body2 typography]

[Tag chip] [Tag chip] [Tag chip] ...
```

- Tags sourced from: genres + filtered categories (Steam Store API)
- Language: `schinese` (Simplified Chinese) — matches user's preferred tag language
- Filtered out: Steam Cloud, Remote Play variants, Steam Leaderboards, VAC, Stats, Trading Cards, Workshop
- Tag chips: `Size.Small Variant.Outlined Color.Default`
- Fallback: section hidden if `_storeDetails == null` or both description and tags empty

### Reference Player Panel

Collapsed by default (`IsExpanded="false"`).

Before:
```
[URL input field            ]  [SteamHunters ↗]
[Override burst detection checkbox — long label]
```

After:
```
[URL input field                              ] [ⓘ tooltip checkbox]
```

- Checkbox rendered with no visible label; wrapped in `MudTooltip` with Info icon
- Tooltip text: "Override burst detection — allow scheduling even if reference player unlocked ≥5 achievements at the same timestamp"
- SteamHunters link moved to hero banner (shown contextually when game has achievements)
- Single-row layout reduces visual clutter in the collapsed panel

---

## Visual Consistency Notes

- All new UI uses existing MudBlazor dark-theme CSS variables (`--mud-palette-surface`, `--mud-palette-text-primary`, etc.)
- No new color tokens introduced — chips use `Color.Primary` / `Color.Default` from existing palette
- Responsive breakpoints follow existing `MudGrid xs/sm/md` pattern
- Async store data load uses `InvokeAsync(StateHasChanged)` — standard Blazor Server pattern for background data without blocking
