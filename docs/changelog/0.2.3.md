## v0.2.3

### Fixed
- **Idle stops before all achievements unlock**: `GameIdleService` now marks a game `Completed` only when the time target is reached **and** every achievement row is `IsUnlocked = true`. Previously the loop stopped as soon as `TotalPlayMinutes` hit the target, leaving any achievements with higher `UnlockOffsetMinutes` permanently locked. `targetMinutes` is also computed as `max(TargetHours × 60, ReferencePlayMinutes)` to ensure the scheduler always has enough time to fire.
- **Restarting a completed game idles indefinitely**: If a user manually starts a game that is already `Completed`, the idle loop runs with no time limit (`int.MaxValue`) so it never auto-stops. Since all achievements are already unlocked at that point, the scheduler will not attempt any new unlocks.
- **Timezone setting has no effect on displayed times**: `GameDetail.razor` was using `ToLocalTime()` (server's OS timezone, always UTC in the Docker container) instead of the `DisplayTimezone` saved in `SteamConfig`. Times are now converted via `TimeZoneInfo.ConvertTimeFromUtc()` using the user-selected timezone, applied to both the achievement unlock column and the "Last synced" timestamp.

## v0.2.3

### 修复
- **挂机在成就全解锁前停止**：`GameIdleService` 现在仅在时间目标达到**且**所有成就行均为 `IsUnlocked = true` 时才标记 `Completed`。此前循环一到目标时长就停止，`UnlockOffsetMinutes` 更大的成就因此永远无法被解锁。`targetMinutes` 同时取 `max(TargetHours × 60, ReferencePlayMinutes)`，确保调度器有足够时间触发。
- **重启已完成游戏时持续挂机**：若用户对 `Completed` 状态的游戏手动点击开始，idle 循环将以无时间上限（`int.MaxValue`）运行，不再自动停止。由于此时所有成就已解锁，调度器不会尝试解锁新成就。
- **切换时区后成就时间不变**：`GameDetail.razor` 此前使用 `ToLocalTime()`（取服务器 OS 时区，Docker 容器内固定为 UTC），完全忽略 `SteamConfig` 中保存的 `DisplayTimezone`。现在改用 `TimeZoneInfo.ConvertTimeFromUtc()` 按用户选择的时区转换，成就解锁时间列与"Last synced"时间戳均已修正。
