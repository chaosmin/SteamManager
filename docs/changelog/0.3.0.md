## v0.3.0

### Added
- **Play Queue**: Games can now be added to a play queue via the playlist button on each game card. A background service polls Steam every 5 minutes; after 2 consecutive checks with no active game session, idling automatically starts for the first game in the queue. When the player launches a different game via the Steam client, the idle session is paused and the elapsed time is saved. Once all achievements are unlocked the game is automatically removed from the queue. The queue is displayed as a banner at the top of the Games page with chip controls for manual removal.
- **Scheduler resume support**: `UnlockSchedulerService` now accepts a `resumeFromMinutes` parameter so that a paused queue entry resumes the unlock schedule from where it left off rather than restarting from zero.
- **Current game detection**: `SteamWebApiClient.GetCurrentGameIdAsync` calls `GetPlayerSummaries/v2` to determine which game the player is currently running, enabling the queue service to detect conflicts and pause accordingly.

### Fixed
- **Missing table startup guard**: Added idempotent `CREATE TABLE IF NOT EXISTS` statements for `steam_audit_log` and `play_queue` in `Program.cs` startup, executed after `MigrateAsync()`. This resolves a migration history/table state mismatch on the NAS database where EF Core reported "already up to date" but the tables did not exist, causing runtime errors on both pages and audit logging.

## v0.3.0

### 新增
- **待玩队列**：可通过游戏卡片上的播放列表按钮将游戏加入待玩队列。后台服务每 5 分钟轮询一次 Steam 状态；连续两次检测到账户未在游戏时，自动开始挂机队列中的第一款游戏。玩家通过 Steam 客户端启动其他游戏时，挂机会暂停并保存已累计时长；全成就解锁后自动移出队列。队列以横幅形式展示在 Games 页面顶部，支持通过 Chip 手动移除。
- **挂机调度恢复支持**：`UnlockSchedulerService` 新增 `resumeFromMinutes` 参数，暂停的队列条目恢复时可从已累计时间点继续执行解锁计划，而非从零重新开始。
- **当前游戏检测**：`SteamWebApiClient.GetCurrentGameIdAsync` 通过调用 `GetPlayerSummaries/v2` 获取玩家当前运行的游戏，供队列服务检测冲突并及时暂停挂机。

### 修复
- **启动时缺失数据表**：在 `Program.cs` 启动流程中（`MigrateAsync()` 之后）增加了幂等的 `CREATE TABLE IF NOT EXISTS` 语句，覆盖 `steam_audit_log` 与 `play_queue` 表。修复了 NAS 数据库上 EF Core 迁移历史与实际表状态不一致的问题——迁移器报告"已是最新"但表实际不存在，导致页面加载失败与审计日志报错。
