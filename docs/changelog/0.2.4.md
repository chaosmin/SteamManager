## v0.2.4

### Added
- **Steam interaction audit log**: A new `steam_audit_log` table records every call to external Steam APIs (SteamKit2 `GetUserStats` / `StoreUserStats`, Steam Web API, SteamHunters, Steam Community badge page). Each row captures the source, operation name, app ID, request summary (API keys stripped), success/failure flag, response summary, and duration in milliseconds.

### Fixed
- **Achievement unlock timestamp not refreshed after SAM lock**: `AchievementHandler` now clears the achievement bit (0) before setting it (1) when the bit is already set. Steam only records a new unlock timestamp on a 0→1 transition, so the extra clear step ensures the timestamp is always current even when the achievement was previously locked by an external tool.
- **Synced unlock time used server `UtcNow` instead of Steam's timestamp**: `UnlockSchedulerService.SaveUnlockedAsync` now queries `GetPlayerAchievements` immediately after each unlock and writes Steam's server-recorded Unix timestamp to the database. Falls back to `DateTime.UtcNow` only if the Steam API call fails.

### Changed
- **Games page loads instantly**: Replaced `OnInitializedAsync` (blocking JOIN query) with `OnAfterRenderAsync` so the page skeleton renders immediately. Data is fetched with two lightweight queries — one for games, one `GROUP BY` aggregation for achievement counts — eliminating the full `Include(g => g.Achievements)` join.
- **"100%" filter replaces "Not 100%"**: The achievement filter switch on the Games page is now labeled "100%" and when enabled shows only games where all achievements are unlocked (previously it showed games with at least one achievement remaining).

## v0.2.4

### 新增
- **Steam 交互审计日志**：新增 `steam_audit_log` 表，记录所有外部 Steam API 调用（SteamKit2 的 `GetUserStats` / `StoreUserStats`、Steam Web API、SteamHunters、Steam 社区徽章页）。每行包含来源、操作名称、AppId、请求摘要（自动脱敏 API Key）、成功/失败标志、返回摘要及耗时（毫秒）。

### 修复
- **SAM 上锁后解锁时间戳未刷新**：`AchievementHandler` 现在在发送设置请求（bit=1）前，若检测到该位已为 1，会先发送清除请求（bit=0）。Steam 仅在 0→1 转换时记录新的解锁时间戳，此步骤确保即使成就曾被外部工具上锁，时间戳也始终是最新的。
- **同步写入的解锁时间为服务器本地时间**：`UnlockSchedulerService.SaveUnlockedAsync` 现在在每次解锁后立即调用 `GetPlayerAchievements` 查询 Steam 服务器记录的 Unix 时间戳，并写入数据库。仅当 Steam API 调用失败时才回退到 `DateTime.UtcNow`。

### 变更
- **Games 页面即时渲染**：将 `OnInitializedAsync`（阻塞式 JOIN 查询）改为 `OnAfterRenderAsync`，页面骨架立即渲染，不再白屏等待数据。数据改用两条轻量查询加载——一条读取游戏列表，一条以 `GROUP BY` 聚合成就计数——去除了原有的 `Include(g => g.Achievements)` 全量 JOIN。
- **"100%" 开关替换"Not 100%"**：Games 页面的成就过滤开关更名为"100%"，启用后仅展示所有成就均已解锁的游戏（原逻辑为展示尚未 100% 的游戏）。
