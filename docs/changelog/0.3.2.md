## v0.3.2

### Added
- **Real-player achievement ordering**: `AchievementDataService` now fetches the top-ranked SteamHunters player (via FlareSolverr) and uses Steam Web API (`GetPlayerAchievements`) to derive the actual unlock order for scheduling. Falls back to SteamHunters `localPercentage` ordering when scraping fails.
- **DLC batch detection** (`AchievementIntervalCalculator.CalculateFromPlayerOrderBatched`): detects calendar gaps >30 days between consecutive player unlock timestamps and splits achievements into batches. Each batch is distributed proportionally within its share of `medianCompletionTime`, separated by 60-minute inter-batch gaps.
- **`/profiles/{steamId64}/` URL support**: `AchievementDataService` now recognises both `/id/{username}/` and `/profiles/{steamId64}/` SteamHunters URL formats when reading a cached `ReferenceUrl`, allowing the Steam64 ID to be used directly without a `ResolveVanityURL` round-trip.
- **Sidebar version number**: bottom of the navigation drawer shows the app version, read dynamically from `AssemblyInformationalVersionAttribute` (set via `<Version>` in the `.csproj`).
- **Dashboard portrait cover art**: "Currently Playing" card now shows `library_600x900.jpg` (portrait capsule) instead of `header.jpg` (horizontal banner).
- **GameDetail Offset column**: new column showing the time gap from the previous achievement (`+xh xm`), inserted between Cost and Unlock At. Column order is now: Achievement → Global % → Status → Cost → Offset → Unlock At.

### Fixed
- **Catch-up achievements no longer fire simultaneously**: when overdue achievements are detected on session start, each now waits the same interval as its original offset delta instead of all firing within seconds.
- **ReferenceUrl preserved on fallback**: `AchievementDataService` no longer overwrites a manually-set player URL with the generic `steamhunters.com/apps/{appId}/achievements` fallback. The generic URL is only written when no URL exists at all.
- **Sidebar version number stripped of git commit hash**: `NavMenu` now calls `.Split('+')[0]` on `InformationalVersion` to remove the SemVer build metadata suffix that .NET Source Link appends (e.g. `0.3.2+fc506cf...` → `0.3.2`).

### Changed
- **Release process includes version bump step**: `CLAUDE.md` now explicitly lists updating `<Version>` in `SteamManager.Web.csproj` as step 2 of the release process.
- **Achievement interval calculation uses real player timestamps**: replaced `CalculateFromPlayerOrderBatched` (even distribution across `medianCompletionTime`) with `Calculate()` (median of actual unlock timestamp intervals per player). Steam timestamps are wall-clock time, so intervals now reflect the real calendar gaps between the reference player's unlocks.

### Docs
- **UI/UX optimization plan**: `docs/ui&ux/20260707.md` — 10-item improvement roadmap covering Bento Grid layout, skeleton loading, empty states, card hover effects, filter polish, achievement table redesign, nav indicator, page transitions, and contrast audit.

## v0.3.2

### 新增
- **基于真实玩家的成就排序**：`AchievementDataService` 通过 FlareSolverr 获取 SteamHunters 排名第一的玩家，再通过 Steam Web API（`GetPlayerAchievements`）拿到该玩家的实际解锁顺序，以此规划解锁计划。抓取失败时回退到 SteamHunters `localPercentage` 排序。
- **DLC 批次检测**（`AchievementIntervalCalculator.CalculateFromPlayerOrderBatched`）：检测玩家解锁时间戳中超过 30 天的日历间隔，将成就拆分为多个批次，每个批次在 `medianCompletionTime` 中按比例分配时间，批次间保留 60 分钟间隔。
- **支持 `/profiles/{steamId64}/` URL 格式**：读取缓存的 `ReferenceUrl` 时，同时识别 `/id/{username}/` 和 `/profiles/{steamId64}/` 两种格式，可直接使用 Steam64 ID 无需额外调用 `ResolveVanityURL`。
- **侧边栏版本号**：导航栏底部展示应用版本号，动态读取自 `AssemblyInformationalVersionAttribute`（由 `.csproj` 中的 `<Version>` 写入）。
- **Dashboard 竖版封面**：「Currently Playing」卡片现在显示 `library_600x900.jpg`（竖版封面），替换原来的 `header.jpg`（横版横幅）。
- **GameDetail Offset 列**：新增列，显示距上一个成就的时间间隔（`+xh xm`），插入 Cost 与 Unlock At 之间。列顺序调整为：Achievement → Global % → Status → Cost → Offset → Unlock At。

### 修复
- **catch-up 成就不再同时触发**：会话启动时检测到过期成就，每个成就现在等待与原排期相同的间隔后再解锁，不再在数秒内全部触发。
- **ReferenceUrl 在回退时保留**：不再将手动设置的玩家 URL 覆盖为通用 SteamHunters 链接，只有在完全没有 URL 时才写入通用链接。
- **侧边栏版本号去除 git commit hash**：`NavMenu` 对 `InformationalVersion` 调用 `.Split('+')[0]`，移除 .NET Source Link 追加的 SemVer 构建元数据后缀（如 `0.3.2+fc506cf...` → `0.3.2`）。

### 变更
- **发布流程新增版本号更新步骤**：`CLAUDE.md` 明确将更新 `SteamManager.Web.csproj` 中的 `<Version>` 列为发布流程第 2 步。
- **成就间隔计算改用真实玩家时间戳**：将 `CalculateFromPlayerOrderBatched`（均匀分布到 `medianCompletionTime`）替换为 `Calculate()`（取各玩家实际解锁时间戳间隔的中位数）。Steam 时间戳是日历时间，因此间隔现在反映参考玩家解锁之间的真实日历间隔。

### 文档
- **UI/UX 优化方案**：`docs/ui&ux/20260707.md` — 10 项改进路线图，涵盖 Bento Grid 布局、骨架屏、空状态、卡片悬停动效、筛选折叠、成就表格优化、导航指示器、页面过渡和对比度审查。
