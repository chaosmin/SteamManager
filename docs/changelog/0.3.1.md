## v0.3.1

### Fixed
- **Achievement scheduler uses delta-based timer**: `UnlockSchedulerService` completely rewritten — achievement fires when `elapsed_since_last_unlock ≥ offset_diff`, independent of `TotalPlayMinutes`. Previous implementation incorrectly waited the full offset from session start instead of the gap since the last unlocked achievement.
- **Startup recovery restores saved delta**: `StartupRecoveryService` was passing `TotalPlayMinutes` as the resume offset, causing excessively long waits on restart. Now relies entirely on DB-persisted `SavedIdleDeltaMinutes`.
- **Dashboard "Next Achievement" skipped achievements**: Filter `UnlockOffsetMinutes > TotalPlayMinutes` caused the next achievement to disappear when a Steam sync updated `TotalPlayMinutes` beyond the achievement's offset. Now shows the first locked achievement by offset regardless of playtime.
- **Dashboard countdown uses live scheduler timer**: Countdown was `nextAch.UnlockOffsetMinutes − TotalPlayMinutes`; now calls `SchedulerService.GetTimeUntilNext()` which reads directly from the in-memory session start time.

### Added
- **Per-game timer persistence** (`Game.SavedIdleDeltaMinutes`): idle progress is saved to DB on scheduler stop and restored on next start, so server restarts and disconnects no longer reset the unlock schedule.
- **Dashboard upcoming achievements row**: A second row in "Currently Playing → Next Achievement" shows all remaining pending achievement icons in order. Hovering each icon shows the achievement name and live countdown, refreshed every minute.
- **Server-validated achievement badge**: Game cards show an orange shield icon when `steam_audit_log` contains a failed `StoreUserStats` operation for that game, indicating achievements cannot be auto-unlocked.
- **`IUnlockSchedulerService.GetTimeUntilNext(appId, offsetMinutes)`**: New interface method for per-achievement live countdown; computes `sessionStart + offset − now` from in-memory scheduler state.

### Changed
- **DB columns renamed to snake_case**: All five tables (`game`, `achievement`, `steam_config`, `play_queue`, `steam_audit_log`) switched from PascalCase to snake_case. EF Core applies the convention globally via `ToSnakeCase()` in `OnModelCreating`.
- **Column order normalized**: `created_at` / `updated_at` moved to the last position in all tables.
- **`TargetHours` removed**: Field was hardcoded to `10h` across all rows and unused by any live logic. Removed from `Game` model and all call sites; fallback replaced with `600` minute literal.
- **100% filter switch color**: Changed from `Color.Secondary` (invisible on dark background) to `Color.Success`.

## v0.3.1

### 修复
- **成就调度器改为增量计时**：`UnlockSchedulerService` 完全重写——成就在「距上次解锁已过时间 ≥ 当前偏移量 − 上次解锁偏移量」时触发，与 `TotalPlayMinutes` 完全无关。原实现错误地从会话开始等待完整偏移量，导致重启后等待时间极长。
- **启动恢复使用已保存增量**：`StartupRecoveryService` 原先将 `TotalPlayMinutes` 作为恢复偏移量传入。现在完全依赖 DB 持久化的 `SavedIdleDeltaMinutes`。
- **Dashboard「下一个成就」跳过成就**：过滤条件 `UnlockOffsetMinutes > TotalPlayMinutes` 在 Steam 同步更新游戏时长超过成就偏移量后会使该成就从 Dashboard 消失。现在不论游戏时长，始终显示偏移量最小的首个未解锁成就。
- **Dashboard 倒计时使用实时调度器时间**：原倒计时为 `成就偏移量 − TotalPlayMinutes`；现调用 `SchedulerService.GetTimeUntilNext()`，直接基于内存中的会话起始时间计算。

### 新增
- **每游戏计时器持久化**（`Game.SavedIdleDeltaMinutes`）：调度器停止时将已挂机增量保存至 DB，下次启动时恢复，服务重启或断线不再重置解锁进度。
- **Dashboard 后续成就图标行**：「Currently Playing → Next Achievement」新增第二行，按顺序展示所有后续待解锁成就的灰色图标，鼠标悬停弹出成就名称与实时倒计时（每分钟刷新）。
- **服务器校验成就标记**：游戏卡片右下角显示橙色盾牌图标，当 `steam_audit_log` 中存在该游戏的 `StoreUserStats` 失败记录时出现，表示成就无法自动解锁。
- **`IUnlockSchedulerService.GetTimeUntilNext(appId, offsetMinutes)`**：新增接口方法，基于内存调度器状态计算各成就实时倒计时。

### 变更
- **DB 列名改为 snake_case**：五张表全部从 PascalCase 改为 snake_case，EF Core 通过 `OnModelCreating` 中的 `ToSnakeCase()` 全局应用。
- **列顺序规范化**：所有含时间戳的表中，`created_at` / `updated_at` 移至末尾。
- **移除 `TargetHours`**：该字段全表硬编码为 `10h` 且无实际用途，已从模型及所有调用点删除，兜底值改为字面量 `600` 分钟。
- **100% 筛选开关颜色**：从 `Color.Secondary`（深色背景不可见）改为 `Color.Success`。
