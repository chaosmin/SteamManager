# v0.4.1 — UI 重设计 & 队列 Bug 修复

## EN

### Bug Fixes

- **StopQueue removed game from queue** — clicking Stop Queue was calling `RemoveFromQueueAsync` (permanent delete). Fixed by adding `PauseQueueAsync`: stops the idle session and resets status to `Idle` without removing the queue entry. Game resumes at its original queue position on next Start.
- **Force Replay not re-queuing** — after Force Refresh on a `Completed` game, `game.Status` remained `Completed`, so `GetQueueAsync` (which filters out Completed) never saw the entry and the game never started. Fixed by resetting `game.Status = GameStatus.Idle` in the `forceReset` block before saving.
- **Queue Progress empty after pause+resume** — `GameIdleService.StopAsync` was clearing `ScheduledUnlockAt = null` on all pending achievements on pause. On resume, `StartGameFromQueueAsync` only re-anchors achievements with an existing `ScheduledUnlockAt`, so after a pause/resume cycle there was nothing to show. Fixed by removing the schedule-clearing loop from `StopAsync` — schedules are preserved through pause and re-anchored correctly on next start.
- **GameDetail time format duplicate year** — `FormatTime` default format was `"yyyy-yyyy-MM-dd HH:mm"` (duplicated `yyyy-`). Fixed to `"yyyy-MM-dd HH:mm"`.

### UI / UX

#### Games Page — Filter Redesign

- Replaced `MudSwitch` toggle row with a chip-based filter system
- **Status chips** (single-select): All / Playing / Queued / Scheduled / Completed — "Queued" matches games in `Idle` state that are in the queue
- **Property chips** (multi-select): Has Achievements / 100% Done / Card Drops
- Result count (`N game(s)`) shown above the grid
- **Clear All** button appears when any filter is active
- Sync button now visible for `Completed` games (useful for refreshing Steam playtime after manual changes)

#### GameDetail Page — Layout Redesign

- **Hero banner** — full-width Steam header image (460:215 aspect ratio) with gradient overlay; title, status chip, Steam Store button, and SteamHunters button (only shown when achievements > 0) overlaid at bottom-left
- **Stats strip** — horizontal card row: Playtime / Target / Achievements / App ID / Last Synced
- **Game description + tags** — short description and genre/category tags fetched from Steam Store API (`store.steampowered.com/api/appdetails`); tags display in Simplified Chinese (`schinese`); loaded asynchronously without blocking page render
- **Reference player panel** — URL input and Override burst checkbox rendered inline in a single row; SteamHunters redirect button removed from this panel (moved to hero); checkbox label replaced with a tooltip on an info icon (text was too long for inline display)

### New Features

- **Steam Store integration** — `SteamWebApiClient.GetAppStoreDetailsAsync` fetches `short_description`, genres, and filtered categories (skips Steam Cloud, Workshop, VAC, leaderboard, trading cards, etc.); result cached in `_storeDetails` on GameDetail load
- **SteamHunters button in hero** — links to `https://steamhunters.com/apps/{appId}`; hidden when no achievements are configured for the game

## 中文

### 修复

- **停止队列移除游戏** — 点击停止队列时调用的是 `RemoveFromQueueAsync`（永久删除队列条目）。新增 `PauseQueueAsync`：停止挂机会话、状态重置为 `Idle`，但不删除队列条目，下次 Start 时可从原位置继续。
- **Force Replay 后游戏不重新入队** — 对 `Completed` 游戏执行 Force Refresh 后，`game.Status` 仍为 `Completed`，而 `GetQueueAsync` 过滤掉 Completed 游戏，导致游戏永远不会启动。修复：在 `forceReset` 块中保存前将 `game.Status` 重置为 `GameStatus.Idle`。
- **暂停后重启队列 Queue Progress 无数据** — `GameIdleService.StopAsync` 在暂停时清空了所有待解锁成就的 `ScheduledUnlockAt`。重新启动时 `StartGameFromQueueAsync` 只重锚定 `ScheduledUnlockAt` 有值的成就，因此暂停/重启后 Queue Progress 无待显示数据。修复：从 `StopAsync` 中移除清空计划时间的循环，暂停时保留计划时间，重启时正常重锚定。
- **GameDetail 时间格式重复年份** — `FormatTime` 默认格式为 `"yyyy-yyyy-MM-dd HH:mm"`（重复了 `yyyy-`）。修复为 `"yyyy-MM-dd HH:mm"`。

### UI / UX

#### Games 页 — 筛选栏重设计

- 用 Chip 体系替换原有的 `MudSwitch` 开关行
- **状态 Chip**（单选）：全部 / Playing / Queued / Scheduled / Completed — "Queued" 匹配 `Idle` 状态且在队列中的游戏
- **属性 Chip**（多选）：Has Achievements / 100% Done / Card Drops
- 网格上方显示结果计数（`N 款游戏`）
- 有筛选条件激活时显示 **清除全部** 按钮
- `Completed` 游戏也显示 Sync 按钮（用于手动更新 Steam 游戏时长）

#### GameDetail 页 — 布局重设计

- **Hero 横幅** — 全宽 Steam 头图（460:215 比例），渐变遮罩；标题、状态 Chip、Steam Store 按钮、SteamHunters 按钮（有成就时才显示）叠加在左下角
- **统计条** — 横向卡片行：游戏时长 / 目标时长 / 成就数 / App ID / 最后同步时间
- **游戏简介 + 标签** — 从 Steam Store API 获取简短描述和分类标签，标签以简体中文显示，异步加载不阻塞页面渲染
- **参考玩家面板** — URL 输入框与覆盖数据质量检测复选框合并为单行；SteamHunters 跳转按钮从此面板移除（移至 Hero）；复选框标签改为悬停提示（原文本过长）

### 新功能

- **Steam Store 数据集成** — `SteamWebApiClient.GetAppStoreDetailsAsync` 拉取 `short_description`、游戏类型和经过过滤的分类（排除 Steam Cloud、创意工坊、VAC、排行榜、集换式卡片等）；GameDetail 加载时异步缓存至 `_storeDetails`
- **Hero 中的 SteamHunters 按钮** — 链接到 `https://steamhunters.com/apps/{appId}`；游戏无成就数据时隐藏
